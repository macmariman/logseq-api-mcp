> **Status:** Draft pending user approval.

## 1. Goal

Close every gap identified in the v1.0.0 audit cross-checked against:

- [Logseq HTTP API contract](https://wiki.jamesravey.me/books/software-engineering/page/logseq-http-api) (`POST /api`, `{method,args}`, Bearer auth, `401/400/404/500`),
- [`IEditorProxy`](https://logseq.github.io/plugins/interfaces/IEditorProxy.html) TypeScript signatures (`createPage`, `appendBlockInPage`, `insertBatchBlock`, etc.),
- [MCP `build-server`](https://modelcontextprotocol.io/docs/develop/build-server) stdio constraint (no stdout writes).

…while honoring the six standards declared in `ROADMAP.md` (efficiency, KISS, immutability, decoupling, agnostic architecture, structured docstrings).

The result is a release that:

1. Is **correct** against Logseq's documented plugin API,
2. **Survives** MCP stdio without corrupting the JSON-RPC stream,
3. Ships **no aspirational stubs** — the vector subsystem is retracted,
4. Has a **shared client/config** wired through the registry, not re-instantiated per call,
5. Documents the breaking-change tolerance via `MIGRATION.md` despite the patch-version label.

### 4.1 Registry wiring (decoupling, std. #4)

`register_all_tools` signature:

```python
def register_all_tools(mcp_server: FastMCP, client: LogseqClient, config: LogseqConfig) -> None:
    """Register every discovered tool with the FastMCP server.

    @param mcp_server FastMCP instance whose .tool() decorator is invoked.
    @param client     Shared LogseqClient instance; one aiohttp.ClientSession lifecycle for the whole process.
    @param config     Shared immutable LogseqConfig (verify_ssl, db_mode, exclude_tags, …).
    @returns None
    @throws Nothing (import failures are logged to stderr via the discovery layer).
    @complexity O(T) where T = tool count.
    """
```

The public tool wrapper is no longer a closure that calls `load_config()`. Instead each `_run` becomes the registered callable via `functools.partial(_run, client, config)`. The FastMCP `@tool` decorator inspects `partial`'s wrapped function signature for the JSON schema; we verify this works in the registry test.

### 4.2 Error mapping (std. #3, #5)

`_call`:

```python
async def _call(self, method: str, args: list | None = None) -> object:
    """Issue a JSON-RPC POST to /api and map HTTP status to typed exceptions.

    @param method  Logseq method name, e.g. "logseq.Editor.getAllPages".
    @param args    Positional args list forwarded as JSON.
    @returns       Parsed JSON response (object | list | None).
    @throws LogseqAuthError       401.
    @throws LogseqNotFoundError   404.
    @throws LogseqAPIError        any other 4xx/5xx with status_code populated.
    @throws LogseqConnectionError aiohttp.ClientConnectorError, asyncio.TimeoutError.
    @complexity O(1) network call.
    """
```

### 4.3 `update_page` (std. #1, #2, #3)

```python
async def _run(client, config, page_name, content=None, mode="append", properties=None):
    """Update a Logseq page.

    @complexity O(B) where B = parsed top-level block count;
                replace mode adds O(E) for E = existing block count.
    """
    if not content and not properties:
        return _error("Nothing to update: provide content or properties")

    page = await client.get_page(page_name)
    if not page:
        return _error(f"Page '{page_name}' not found")

    merged_props, batch_blocks = _build_payload(content, properties)
    if merged_props:
        await client.set_page_properties(page_name, merged_props)

    if mode == "replace":
        await _clear_existing_blocks(client, page_name)

    if batch_blocks:
        await client.insert_batch_block(page["uuid"], batch_blocks, sibling=False)

    return _success(page_name, mode, batch_blocks, merged_props)
```

`_build_payload`, `_clear_existing_blocks`, `_error`, `_success` are pure helpers in the same module — keeps depth ≤ 1.

## 5. Breaking changes (documented in `MIGRATION.md`)

| Change | Caller-facing effect | Mitigation |
|---|---|---|
| Drop `get_page_links` tool | MCP clients that registered the alias must call `get_page_backlinks`. | `MIGRATION.md` table. |
| `verify_ssl` default → `True` | Direct `LogseqConfig(endpoint, token)` users now verify TLS. | Set `verify_ssl=False` explicitly if needed. |
| `LogseqConnectionError` no longer wraps unknown exceptions | Catchers relying on broad catch will see `LogseqAPIError` (or original `aiohttp.ClientError`). | `except LogseqAPIError` is the right catch now. |
| `create_page` JSON wire shape changed | Older Logseq versions that accepted the packed form should still work; newer versions match the TS signature. | Note minimum supported Logseq version in README. |
| Vector tools removed | `vector_search`, `vector_db_status`, `logseq-sync` no longer exist. | Note retraction in CHANGELOG; point at a future v1.1 vector branch if desired. |
| `pathlib` and `load-dotenv` removed | Downstream `pip install`s no longer pull these. | `python-dotenv` is a drop-in import-compatible replacement (`from dotenv import load_dotenv`). |

The user explicitly chose the `v1.0.1` label despite these breaking changes; the rationale and migration steps live in `MIGRATION.md`.

## 6. Standards compliance audit (each fix vs the six standards)

| Standard | How we honor it |
|---|---|
| **#1 Efficiency** | `resolve_page_uuids` batched from O(U) HTTP to O(1); `excluded_page_names` cached with monotonic TTL; everything else stays O(N) or better. Big-O in every public docstring. |
| **#2 KISS** | Every `_run` follows guard-clause → action → result. Nested helpers in `update_page`, `update_page` replace flow, parser frontmatter are extracted as pure module-level functions, not closures. Conditional depth ≤ 1 verified by review. |
| **#3 Pure functions / immutability** | `filter_pages` returns new lists; `extract_tags` returns new lists; parser produces `ParsedContent` without mutating input string; `_build_payload` returns `(dict, list[dict])`. No tool mutates `client` or `config`. |
| **#4 Decoupling / DI** | `register_all_tools(mcp, client, config)` injects both. `_run(client, config, …)` accepts them as parameters. Tools never call `load_config()` or instantiate `LogseqClient`. |
| **#5 Testability** | `FakeLogseqClient` already in `tests/conftest.py`; we extend its method list to match the new client surface (batched `datascript_query`, `excluded_page_names`). No new `aiohttp` patching outside `tests/client/`. |
| **#6 Docstrings** | Every public function gains `@param`/`@returns`/`@throws`/`@complexity`. The roadmap's existing convention (`Args:/Returns:/Raises:/Complexity:`) is retained for consistency with the codebase — semantically equivalent. |

## 9. Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| `create_page` arg-shape fix breaks against old Logseq builds (<0.10.x) | Medium | Document minimum Logseq version in README; keep a feature-flag escape via env var if a tester reports breakage. |
| Subagents diverge on shared interfaces between tasks | Medium | The writing-plans output pre-declares every signature in Task 0 (a "lock" task). Each subsequent subagent reads `tasks_completed` plus the design doc. |
| Removing vector breaks downstream users who already installed `[vector]` | Low | CHANGELOG + MIGRATION + retain `[dependency-groups]` empty stub to avoid `uv sync --group vector` hard-failing during the migration cycle. |
| `MIGRATION.md` claims for `python-dotenv` import-compat are wrong | Low | Grep for `from load_dotenv import …` / `import load_dotenv` first; both `python-dotenv` and `load-dotenv` ship `from dotenv import load_dotenv` so substitution is safe — verified before the swap. |
| TLS verification flip breaks local dev where users have self-signed certs | Low | `LOGSEQ_VERIFY_SSL=false` env var still honored; documented in README + MIGRATION. |
